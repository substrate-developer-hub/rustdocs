Canonical hash trie definitions and helper functions.

Each CHT is a trie mapping block numbers to canonical hash. One is generated for every SIZE blocks, allowing us to discard those blocks in favor of the trie root. When the "ancient" blocks need to be accessed, we simply request an inclusion proof of a specific block number against the trie with the root has. A correct proof implies that the claimed block is identical to the one we discarded.

Get the starting block of a given CHT. CHT 0 includes block 1...SIZE, CHT 1 includes block SIZE + 1 ... 2SIZE More generally: CHT N includes block (1 + NSIZE)...((N+1)*SIZE). This is because the genesis hash is assumed to be known and including it would be redundant.
